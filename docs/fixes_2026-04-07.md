本次修复主要解决了以下问题：

## 1. graphiti_service.py

### 问题
- MiniMax LLM 返回的边数据使用 `source_entity_name`/`target_entity_name` 字段名，而非 Graphiti 期望的 `source_node_name`/`target_node_name`
- MiniMax LLM 返回实体摘要时使用 Markdown 格式 `**Name**: "summary"` 而非 JSON

### 修复内容

#### 1.1 边字段名转换 (第 50-67 行)
```python
# ExtractedEdges 格式: {"edges": [...]} 或直接数组
elif 'ExtractedEntities' in model_name or 'Edge' in str(response_model.model_fields):
    if isinstance(parsed, list):
        # MiniMax 返回的是数组，需要转换字段名
        converted_edges = []
        for edge in parsed:
            if isinstance(edge, dict):
                converted_edge = {}
                if 'source_entity_name' in edge:
                    converted_edge['source_node_name'] = edge['source_entity_name']
                if 'target_entity_name' in edge:
                    converted_edge['target_node_name'] = edge['target_entity_name']
                if 'relation_type' in edge:
                    converted_edge['name'] = edge['relation_type']
```

#### 1.2 Markdown 摘要格式解析 (第 100-110 行)
```python
# 方法3.5: 处理 **Name**: "summary" 格式的 Markdown 摘要响应
if '**' in result and '": "' in result:
    log(f'检测到 **Name**: "summary" 格式的 Markdown 摘要')
    summaries = []
    summary_pattern = r'\*\*([^*]+)\*\*:\s*"([^"]*)"'
    for match in re.finditer(summary_pattern, result):
        entity_name = match.group(1).strip()
        entity_summary = match.group(2).strip()
        summaries.append({"name": entity_name, "summary": entity_summary})
```

## 3. __init__.py (services 模块)

### 问题

### 修复内容
使所有 Zep 相关导入成为可选导入：

```python
try:
    from .zep_entity_reader import ZepEntityReader, EntityNode, FilteredEntities
    from .zep_graph_memory_updater import ZepGraphMemoryUpdater, ZepGraphMemoryManager, AgentActivity
    HAS_ZEP = True
except ImportError:
    HAS_ZEP = False
    ZepEntityReader = None
    EntityNode = None
    FilteredEntities = None
    ZepGraphMemoryUpdater = None
    ZepGraphMemoryManager = None
```
